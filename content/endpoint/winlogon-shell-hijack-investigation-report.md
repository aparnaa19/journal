# Winlogon Shell Hijack Investigation Report

Alert Summary</strong></p><p>A malicious executable a.exe was detected on endpoint IEWIN7 modifying the Winlogon Shell registry key. The investigation confirms persistent access was established by injecting BRE6BgE2JubB.exe into the Winlogon Shell, causing it to execute silently on every user login before the desktop loads.</p><p><strong>2. Investigation Steps</strong></p><p><strong>Dataset used : </strong><a href="https://github.com/sbousseaden/EVTX-ATTACK-SAMPLES"><strong>https://github.com/sbousseaden/EVTX-ATTACK-SAMPLES</strong></a></p><p><img src="media/winlogon-shell-hijack-investigation-report-1.png" /></p><p><strong>Step 1 : Hunt Query</strong></p><pre><code>winlog.event_id : "13" AND registry.path : *Winlogon*</code></pre><p>Returned 1 result confirming Winlogon Shell value was modified on IEWIN7.</p><p><img src="media/winlogon-shell-hijack-investigation-report-2.png" /></p><p><strong>Step 2 : Event Analysis</strong></p><p>Key findings:</p><ul><li>Dropper: C:\Users\IEUser\Downloads\a.exe executed with Process ID 4020</li><li>Registry key modified: HKU\...\CurrentVersion\Winlogon\Shell</li><li>Data written: "C:\Users\IEUser\AppData\Roaming\9QxTsAU9w8gyPj4w\BRE6BgE2JubB.exe", explorer.exe</li><li>Normal Winlogon Shell value should be explorer.exe only</li><li>Sysmon rule Persistence - Winlogon Shell was triggered automatically</li></ul><p>Red flags:</p><ul><li>a.exe running from C:\Users\IEUser\Downloads\ - legitimate software never executes from Downloads</li><li>9QxTsAU9w8gyPj4w is a randomly generated folder name - classic malware behavior</li><li>BRE6BgE2JubB.exe is a randomly named executable - designed to avoid recognition</li><li>Winlogon Shell modified to run BRE6BgE2JubB.exe BEFORE explorer.exe - user sees normal desktop while malware runs silently</li><li>HKU hive - affects the specific user account IEUser</li></ul><p><img src="media/winlogon-shell-hijack-investigation-report-3.png" /></p><p><img src="media/winlogon-shell-hijack-investigation-report-4.png" /></p><p><strong>Step 3 : Attack Reconstruction</strong></p><p>Attacker gains access to IEWIN7</p><p>↓</p><p>a.exe dropped in C:\Users\IEUser\Downloads\</p><p>↓</p><p>a.exe executed (Process ID: 4020)</p><p>↓</p><p>BRE6BgE2JubB.exe dropped in C:\Users\IEUser\AppData\Roaming\9QxTsAU9w8gyPj4w\</p><p>↓</p><p>Winlogon Shell registry key modified:</p><p>HKU\...\Winlogon\Shell = "BRE6BgE2JubB.exe", explorer.exe</p><p>↓</p><p>Every IEUser login → BRE6BgE2JubB.exe runs silently first</p><p>↓</p><p>explorer.exe loads normally - user sees nothing unusual</p><p>↓</p><p>Attacker maintains persistent access</p><p><strong>Step 4 : Persistence Confirmed?</strong></p><p>Yes. Winlogon Shell modification confirmed via Sysmon Event ID 13. BRE6BgE2JubB.exe will execute automatically on every login of the IEUser account until the registry value is restored to explorer.exe only.</p><p><strong>4. Conclusion</strong></p><p>The endpoint IEWIN7 was compromised via Winlogon Shell hijacking (T1547.004). The dropper a.exe executed from the Downloads folder, dropped a randomly named payload BRE6BgE2JubB.exe in a randomly named AppData folder, and modified the Winlogon Shell registry key to achieve persistence. This technique is stealthier than Run key persistence because the malware executes before the desktop loads, making it harder for the user to notice.</p><p><strong>5. Recommended Actions</strong></p><ul><li>Isolate IEWIN7 immediately</li><li>Restore Winlogon Shell registry value to explorer.exe only</li><li>Delete C:\Users\IEUser\AppData\Roaming\9QxTsAU9w8gyPj4w\ folder and all contents</li><li>Delete C:\Users\IEUser\Downloads\a.exe</li><li>Investigate how a.exe was initially delivered to the Downloads folder</li><li>Check all other user accounts on IEWIN7 for similar Winlogon modifications</li><li>Hunt for BRE6BgE2JubB.exe or 9QxTsAU9w8gyPj4w folder on other machines</li></ul><p><strong>7.
