# Registry Run Key Persistence Investigation Report
display_title: Registry Run Key Persistence Investigation Report
skills: Windows registry forensics, Sysmon Event ID 13 analysis, registry run key persistence detection, Winlogon Shell hijack detection, EVTX log parsing, KQL threat hunting queries, IOC verification, MITRE ATT&CK technique mapping (T1547.001, T1547.004, T1036), masquerading detection, malicious executable path analysis, 
tools: Elastic SIEM, Kibana, Elasticsearch, Winlogbeat, Docker, Python, evtx library, KQL, Sysmon, EVTX-ATTACK-SAMPLES dataset, MITRE ATT&CK Navigator

<p><strong>Severity:</strong> Critical </p><p><strong>Status:</strong> Confirmed Malicious </p><p><strong>MITRE ATT&amp;CK:</strong> T1547.001, T1036</p><p><strong>1. Alert Summary</strong></p><p>A malicious executable a.exe was detected on endpoint MSEDGEWIN10 modifying the HKLM Run registry key. The investigation confirms persistent access was established for all users by disguising the malware path as a legitimate Windows process taskhost.exe.</p><p><strong>2. Affected System</strong></p><table><tr><td><p><strong>Field</strong></p></td><td><p><strong>Value</strong></p></td></tr><tr><td><p>Hostname</p></td><td><p>MSEDGEWIN10</p></td></tr><tr><td><p>Attack Time</p></td><td><p>2020-07-04 14:18:58 UTC</p></td></tr><tr><td><p>Malicious File</p></td><td><p>a.exe</p></td></tr><tr><td><p>File Location</p></td><td><p>C:\Users\Public\tools\evasion\</p></td></tr><tr><td><p>Registry Key</p></td><td><p>HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\</p></td></tr><tr><td><p>Data Written</p></td><td><p>"c:\windows\tasks\taskhost.exe"</p></td></tr><tr><td><p>Process ID</p></td><td><p>3728</p></td></tr><tr><td><p>Rule Triggered</p></td><td><p>Persistence - Hidden Run value detected</p></td></tr></table><p><strong>3. Investigation Steps</strong></p><p><strong>All the logs were ingested from the dataset – “https://github.com/sbousseaden/EVTX-ATTACK-SAMPLES”</strong></p><p><img src="media/registry-run-key-persistence-investigation-report-1.png" /></p><p><strong>Step 1 : Hunt Query</strong></p><pre><code>winlog.event_id : "13" AND registry.path : *CurrentVersion\\Run*</code></pre><p>Returned 1 result confirming a registry value was set under the HKLM Run key on MSEDGEWIN10.</p><p><img src="media/registry-run-key-persistence-investigation-report-2.png" /></p><p><strong>Step 2 : Event Analysis</strong></p><p>Key findings:</p><ul><li>Malicious file: C:\Users\Public\tools\evasion\a.exe executed with Process ID 3728</li><li>Registry key modified: HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\</li><li>Data written: "c:\windows\tasks\taskhost.exe"</li><li>HKLM affects ALL users on the machine and not just one account</li><li>Sysmon rule Persistence - Hidden Run value detected triggered automatically</li></ul><p>Red flags:</p><ul><li>C:\Users\Public\ is world-writable. Any user or malware can place files here</li><li>Subfolder named evasion explicitly signals attacker intent</li><li>a.exe is a single letter name designed to be overlooked in process lists</li><li>Data value c:\windows\tasks\taskhost.exe is a fake path - C:\Windows\Tasks\ is empty on Windows</li><li>Real taskhost.exe lives in C:\Windows\System32\ - not Tasks folder</li><li>HKLM Run key is a malware that executes on every user login on every account</li></ul><p><img src="media/registry-run-key-persistence-investigation-report-3.png" /></p><p><strong>Step 3  Attack Reconstruction</strong></p><p>Attacker gains access to MSEDGEWIN10</p><p>↓</p><p>a.exe dropped in C:\Users\Public\tools\evasion\</p><p>↓</p><p>a.exe executed (Process ID: 3728)</p><p>↓</p><p>HKLM Run key modified:</p><p>HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\  -&gt; "c:\windows\tasks\taskhost.exe"</p><p>↓</p><p>Every user login → Windows reads Run key → executes a.exe</p><p>↓</p><p>a.exe disguised as taskhost.exe - analyst may overlook it</p><p>↓</p><p>Attacker maintains persistent access across all users and reboots</p><p><strong>Step 4 : Persistence Confirmed?</strong></p><p>Yes. HKLM Run key modification confirmed via Sysmon Event ID 13. a.exe will execute automatically on every user login until the registry entry is removed.</p><p><strong>4. Conclusion</strong></p><p>The endpoint MSEDGEWIN10 was compromised via Registry Run Key persistence (T1547.001). The malicious executable a.exe was placed in a publicly accessible folder and registered under the HKLM Run key disguised as taskhost.exe (T1036 - Masquerading). Because HKLM affects all users, this persistence mechanism gives the attacker access regardless of which user logs in, surviving reboots until explicitly removed.</p><p><strong>5. Recommended Actions</strong></p><ul><li>Isolate MSEDGEWIN10 immediately</li><li>Delete registry entry from HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\</li><li>Delete C:\Users\Public\tools\evasion\a.exe and the entire evasion folder</li><li>Investigate how a.exe was initially delivered to the Public folder</li><li>Check Event ID 4688 for any processes spawned by a.exe</li><li>Hunt for a.exe or evasion folder on other machines in the environment</li><li>Reset credentials for all accounts that logged into MSEDGEWIN10</li></ul><p><strong>7.
