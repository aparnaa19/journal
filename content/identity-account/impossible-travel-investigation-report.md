Alert Summary</strong></p><p>In this investigation, a KQL-based analytics rule running on Microsoft Sentinel detected four user accounts logging in from two geographically distant countries within impossibly short time windows. The shortest gap was 5 minutes between New York and London - a journey that takes a minimum of 7 hours by air. The pattern across all four accounts is consistent with credential theft, where an attacker operating from a foreign location uses stolen credentials while the legitimate user continues normal activity from their home location.</p><p><strong>2. A 333-minute gap is physically impossible.</p><p><img src="media/impossible-travel-investigation-report-2.webp" /></p><p><strong>Step 3 - Detect All Suspicious Accounts Automatically</strong></p><p>In this step, instead of checking each user manually, I built a query to automatically flag any user whose login country changed between consecutive sessions:</p><pre><code>ImpossibleTravel_CL
| order by username_s asc, timestamp_t asc
| extend prev_country = prev(country_s, 1)
| extend prev_city = prev(city_s, 1)
| extend prev_user = prev(username_s, 1)
| extend prev_time = prev(timestamp_t, 1)
| extend time_diff_minutes = datetime_diff('minute', timestamp_t, prev_time)
| where username_s == prev_user
| where country_s != prev_country
| project timestamp_t, prev_time, time_diff_minutes, username_s, prev_city, city_s, prev_country, country_s</code></pre><p><strong>Results - All 4 flagged accounts:</strong></p><table><thead><tr><th><p><strong>Username</strong></p></th><th><p><strong>Previous City</strong></p></th><th><p><strong>Current City</strong></p></th><th><p><strong>Time Gap (mins)</strong></p></th><th><p><strong>Verdict</strong></p></th></tr></thead><tbody><tr><td><p>frank.miller</p></td><td><p>Chicago 🇺🇸</p></td><td><p>Tokyo 🇯🇵</p></td><td><p>10</p></td><td><p>Impossible - 6,300 miles in 10 mins</p></td></tr><tr><td><p>grace.lee</p></td><td><p>New York 🇺🇸</p></td><td><p>London 🇬🇧</p></td><td><p>5</p></td><td><p>Impossible - 3,450 miles in 5 mins</p></td></tr><tr><td><p>henry.grant</p></td><td><p>Moscow 🇷🇺</p></td><td><p>Chicago 🇺🇸</p></td><td><p>333</p></td><td><p>Impossible - 5,000 miles in 5.5 hrs</p></td></tr><tr><td><p>irene.ngo</p></td><td><p>Los Angeles 🇺🇸</p></td><td><p>Sydney 🇦🇺</p></td><td><p>8</p></td><td><p>Impossible - 7,500 miles in 8 mins</p></td></tr></tbody></table><p><img src="media/impossible-travel-investigation-report-3.webp" /><img src="media/impossible-travel-investigation-report-4.webp" /></p><p><strong>Step 4 - Analytics Rule Created</strong></p><p>In this step, I created a scheduled analytics rule in Microsoft Sentinel to automate this detection going forward.
