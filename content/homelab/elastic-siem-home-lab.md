<p>Elastic SIEM + Winlogbeat </p><p><strong>1. Lab Overview</strong></p><p>This home lab simulates a Security Operations Center (SOC) environment for hands-on investigation of real-world attack logs. The setup uses Elastic SIEM running in Docker on a local Windows machine, with Winlogbeat and a custom Python script shipping Windows event logs into Elasticsearch for analysis in Kibana.</p><p><strong>2. Lab Architecture</strong></p><table><tr><td><p><strong>Component</strong></p></td><td><p><strong>Tool</strong></p></td><td><p><strong>Purpose</strong></p></td></tr><tr><td><p>SIEM Platform</p></td><td><p>Elastic + Kibana v8.13.0</p></td><td><p>Log ingestion, search, and visualisation</p></td></tr><tr><td><p>Log Collector</p></td><td><p>Winlogbeat v9.4.2</p></td><td><p>Ships Windows event logs into Elasticsearch</p></td></tr><tr><td><p>Log Collector (EVTX)</p></td><td><p>Python + evtx + elasticsearch libraries</p></td><td><p>Direct EVTX file ingestion into Elasticsearch</p></td></tr><tr><td><p>Dataset</p></td><td><p>EVTX-ATTACK-SAMPLES</p></td><td><p>Real-world attack log samples (.evtx files)</p></td></tr><tr><td><p>Container Runtime</p></td><td><p>Docker Desktop</p></td><td><p>Runs Elasticsearch and Kibana locally</p></td></tr><tr><td><p>Host OS</p></td><td><p>Windows 11</p></td><td><p>Main machine running the lab</p></td></tr></table><p><strong>3. Prerequisites</strong></p><ul><li>Windows 11 (Home or Pro)</li><li>Docker Desktop installed and running</li><li>Winlogbeat v9.4.2 downloaded and extracted</li><li>Python 3.x with evtx and elasticsearch libraries installed</li><li>EVTX-ATTACK-SAMPLES dataset downloaded and extracted from GitHub</li><li>Windows Defender real-time protection temporarily disabled during extraction</li></ul><p><strong>4. Tools and Versions</strong></p><table><tr><td><p><strong>Tool</strong></p></td><td><p><strong>Version</strong></p></td><td><p><strong>Purpose</strong></p></td></tr><tr><td><p>Elasticsearch</p></td><td><p>8.13.0</p></td><td><p>Log storage and search engine</p></td></tr><tr><td><p>Kibana</p></td><td><p>8.13.0</p></td><td><p>SIEM interface and visualisation</p></td></tr><tr><td><p>Winlogbeat</p></td><td><p>9.4.2</p></td><td><p>Windows event log shipper</p></td></tr><tr><td><p>Python</p></td><td><p>3.x</p></td><td><p>Script runtime for EVTX ingestion</p></td></tr><tr><td><p>evtx (Python library)</p></td><td><p>0.11.1</p></td><td><p>Parses .evtx files</p></td></tr><tr><td><p>elasticsearch (Python library)</p></td><td><p>8.x</p></td><td><p>Sends parsed logs to Elasticsearch</p></td></tr><tr><td><p>Docker Desktop</p></td><td><p>Latest</p></td><td><p>Container runtime</p></td></tr></table><p><strong>5. Setup Steps</strong></p><p><strong>Step 1 - Deploy Elasticsearch and Kibana using Docker</strong></p><p>Elasticsearch and Kibana are deployed using Docker Compose. A docker-compose.yml file is created in a dedicated folder and both containers are started using the following command in PowerShell:</p><pre><code>cd "E:\softwares and tools\elastic"
docker-compose up -d</code></pre><p>The command pulls the Elasticsearch and Kibana images and starts both containers. Elasticsearch runs on port 9200 and Kibana runs on port 5601.</p><p><img src="media/elastic-siem-home-lab-1.png" /></p><p>After both containers start, the kibana_system user password must be set so Kibana can authenticate with Elasticsearch. This is done using the following PowerShell command:</p><pre><code>Invoke-WebRequest -Uri "http://localhost:9200/_security/user/kibana_system/_password" -Method POST -Headers @{"Content-Type"="application/json"} -Body '{"password":"elastic123"}' -Credential (New-Object PSCredential("elastic", (ConvertTo-SecureString "elastic123" -AsPlainText -Force)))</code></pre><p><strong><img src="media/elastic-siem-home-lab-2.png" /></strong></p><p><strong>Step 2 - Verify Elasticsearch is Running</strong></p><p>Elasticsearch can be verified by opening a browser and navigating to <a href="http://localhost:9200">http://localhost:9200</a>. The browser will prompt for credentials. After entering the elastic username and password, a JSON response showing cluster information confirms Elasticsearch is running correctly.</p><p><strong>Step 3 -- Access Kibana</strong></p><p>After waiting approximately 3 minutes for Kibana to finish starting, navigate to <a href="http://localhost:5601">http://localhost:5601</a> in the browser. The Kibana login page will appear. Log in using the elastic username and password.</p><p><strong><img src="media/elastic-siem-home-lab-3.png" /></strong></p><p><strong>Step 4 -- Install Python Libraries</strong></p><p>The following Python libraries are required for EVTX file ingestion. Install them using pip in PowerShell:</p><pre><code>pip install evtx elasticsearch</code></pre><p><strong>Step 5 -- Ingest EVTX Files Using Python Script</strong></p><p>For investigations that require specific EVTX attack sample files, a custom Python script called ingest_evtx.py is used. The script reads EVTX files, parses the XML event data, maps key fields to Elastic Common Schema (ECS) format, and sends each event to a dedicated Elasticsearch index called evtx-attack-samples.</p><p>The script is run from PowerShell as follows:</p><pre><code>cd "E:\appu project\endpoint\ingestscript"
python ingest_evtx.py</code></pre><p><strong><img src="media/elastic-siem-home-lab-4.png" />Step 6 -- Configure Winlogbeat for Live Windows Event Logs</strong></p><p>Winlogbeat is used to ingest live Windows event logs and specific EVTX files into Elasticsearch. The key configuration in winlogbeat.yml is as follows:</p><pre><code>yaml
winlogbeat.event_logs:
  - name: 'path\to\your\evtx\file.evtx'
output.elasticsearch:
  hosts: ["http://localhost:9200"]
  username: "elastic"
  password: "elastic123"
  ssl.verification_mode: none
Winlogbeat is launched from PowerShell as Administrator using the following command:
.\winlogbeat.exe -e -c winlogbeat.yml</code></pre><p><strong>Step 7 -- Create Data Views in Kibana</strong></p><p>After logs are ingested, Data Views are created in Kibana to make the data searchable in Discover. To create a Data View, go to Stack Management, then Data Views, then click Create data view.</p><p><strong>8. Restarting the Lab</strong></p><p>After shutting down the machine, use the following steps to restart the lab:</p><ol><li>Open PowerShell and navigate to the elastic folder</li><li>Run docker-compose up -d to start Elasticsearch and Kibana</li><li>Wait 3 minutes then open <a href="http://localhost:5601">http://localhost:5601</a> in browser</li><li>Log in with elastic credentials</li><li>Logs from previous sessions are already in Elasticsearch and do not need to be re-imported</li></ol><p><strong>10. Skills Demonstrated</strong></p><ul><li>Elastic SIEM deployment and configuration using Docker Compose</li><li>Windows Event Log collection using Winlogbeat</li><li>Custom Python scripting for EVTX file ingestion into Elasticsearch</li><li>Log ingestion pipeline setup from Winlogbeat and Python into Elasticsearch and Kibana</li><li>SIEM data view creation and log querying using KQL</li><li>Attack log analysis using real-world EVTX attack samples</li><li>MITRE ATT&amp;CK framework awareness through attack categorisation</li></ul>
